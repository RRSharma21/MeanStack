Experience

Our Experience with The MEAN Stack: 9/10

Difficulty

Learning the syntax for the individual components of the MEAN stack was relatively easy, as they are all written in Javascript, and share similarities to Java in terms of syntax and semantics. The setup for each component was easy and simple, as the MEAN stack website has links explaining how to install each component and make a basic app. The difficult part in working with the MEAN stack was learning how each component interacts with one another.

Learning Curve: MEAN vs HTML/CSS

In Our oppinion the MEAN stack can't be compared to HTML/CSS as HTML/CSS focus more on the presentation of a page or program, thus learning them was not hard in terms of syntax and function, but harder in terms of getting the design to look how we wanted. Whereas the MEAN stack focusses on the functionality of a program/app. In comparison the MEAN stack was much harder, and the majority of our difficulties were encountered in linking every component to one another. Such as using AngularJs to get the task submitted by the user and send it to the database. Or knowing which packages to install with NodeJS's NPM function, or using Express to create the seperate endpoints to link the HTML pages together with the application and server in general.

Remarks

It was fun working with, and learning how to use the MEAN stack as it taught us how to take the knowledge on algorithms and writing programs that we learnt from previous courses and applying it to what we learnt in this course. Doing so allowed us to create a web application that is dynamic and complex but at the same time one that looks simple and clean, something that isn't possible in C or Java. If we would have done anything differently it would have been, trying to add user accounts to our Database so the user could log into their own account and view their specific task list as opposed to viewing the general task list.
